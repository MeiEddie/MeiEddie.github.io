如何搭建本站 ​

VitePress+GitHub托管 ​


选择的理由 ​

搭建网站有许多方法，这只是其中一种，适合自己才是最好的

不选择纯手写html：文章多了难以维护。

VitePress是由Vue官方开发免费、开源、专门做静态文档/博客网站的工具。选择VitePress底层技术纯静态网站，可直接托管GitHub Pages，便于学习使用（免费）。

笔记使用Markdown书写，只要新建.md文件就可以记录学习笔记。单人维护量比较少，维护成本低。

什么是Node.js ​

Node.js可以理解为VitePress的运行环境，VitePress是用JS写的程序，没有node就打不开。

JavaScript是车，nodejs是路，你写的代码跑在nodejs上就相当于造的车跑在路上。有的路是高速公路叫v8，有的路是赛车道叫bun。（来自bilibili@月霞右侍郎）

除了提供运行环境的作用之外，Node.js还包含管理器npm，可一键下载VitePress。

有人或许听过Nest.js，Nest.js是一套基于Node.js的后端开发框架，底层依赖Node.js，貌似使用情景比较少？这里我们不会用到。

从Node.js到VitePress ​

官网下载：https://nodejs.org/zh-cn/download

官网安装有两个选择（安装程序.msi/独立文件.zip），我们选择Windows安装程序 (.msi)，自己选择安装路径。

路径不要带中文、空格（别用D:\我的程序\node这样子的），容易后面npm报错

之后按照安装提示一直点next

不需要勾选Tools for Native Modules的复选框，只用VitePress搭建静态博客，不需要编译 C/C++ 原生模块，这些工具无用，勾选只会增加冗余软件。之后要用到的话也可以安装回来。

新建一个安装文件夹（因为里面的文件是分散的），安装完成之后就能看到下载的文件啦，包括node_modules文件夹和其他各种文件。

先验证Node是否安装成功：

  1. 按下键盘 Win + R，输入 cmd 回车，打开黑色命令窗口
  2. 依次输入下面两条命令，每条输完按回车：
bash
node -v
npm -v

如果输出版本号（比如 v20.15.0、10.8.2）,说明安装成功

新建博客文件夹，初始化VitePress笔记站：

  1. 在电脑非C盘新建文件夹，比如D:\study-blog
  2. 在文件夹地址栏输入cmd回车，直接打开当前目录的终端
  3. 复制执行初始化命令：
bash
npx vitepress init

输入y，Ok to proceed，它一开始要等一会（可能要1~2分钟），一切顺利的话我们就进行下一步

如果等待超过5分钟完全没输出，就是网络超时，按Ctrl+C终止，然后自己想办法（加油）

之后它会问你问题：

Where should VitePress initialize the config?回答./docs（它会给你./）

问你title，description，可以随便写（中英文都可以），不喜欢之后可以改

Theme：选择Default Theme

Use TypeScript for config and theme files? 选择No

Add VitePress npm scripts to package.json? 选择Yes

  1. 安装依赖
bash
npm install
  1. 启动本地博客预览
bash
npm run docs:dev

如果报错：'vitepress'不是内部或外部命令，也不是可运行的程序或批处理文件。说明项目本地的vitepress依赖没有正确安装，在当前对话框内输入：

bash
npm install vitepress --save-dev

出现漏洞警告大概率也没关系，不影响博客正常使用。

运行完之后再次进行步骤5：输入npm run docs:dev

运行成功后，终端会给出地址http://localhost:5173，复制到浏览器就能打开你的网页（在这里点也可以）。

一些疑问 ​

为什么是这个网址

VitePress底层是基于Vite开发的，Vite开发服务器默认端口固定为5173，这是官方设定好的标准端口号，所以只要本地没被占用，启动后一定是这个网址。

旧版Vite曾用3000，所以在一些网上的教程中你会看到网址是http://localhost:3000。

不过当然还是看你的终端怎么显示，复制就行了，终端会显示真实地址。

现在CMD窗口能不能关

取决于现在你是否需要实时编译，现在你在文件夹中更改，浏览器的网页也会实时更新（文件要保存）。关掉CMD窗口网页也会进不去，不需要的情况下可以关掉。想要再次打开只需要在你新建的那个文件夹地址栏输入cmd回车，进行步骤5即可（上文）。

如何开始编辑

.vitepress是配置文件夹，用来改标题、导航栏、侧边栏、搜索、GitHub链接等设置。

index.md文件是首页，可修改首页布局、封面。

api-examples.md / markdown-examples.md是官方示例文档，新手可借此学习，可以删掉。

在docs新建.md格式文件，侧边栏分类需要去到你的config.mjs文件更改，.md格式文件则是你的文章。

Github托管 ​

首先，你需要有一个Github帐号

然后，创建仓库，命名为你的用户名.github.io（要完全一致），选择Public，创建

将你的文件夹内的文件上传GitHub（可使用Git、Github Desktop等等）

注意node_modules文件夹不要上传，文件很大且不需要，影响上传。最好能写一份.gitignore把它隐藏防止操作失误。

自动化部署

  1. 在项目根目录新建文件夹.github/workflows
  2. 在里面新建deploy-pages.yml

文件deploy-pages.yml代码可以参考我的样式：https://github.com/MeiEddie/MeiEddie.github.io

  1. 把新文件也上传仓库

现在你的文件可能大概应该会长这个样子：

YourFile/
├─ .github/
│  └─ workflows/
│     └─ deploy.yml   ← GitHub Actions自动部署脚本
├─ docs/
│  ├─ .vitepress/
│  │  └─ config.mjs  #网站全局配置
│  ├─ index.md  #博客首页
│  ├─ api-examples.md  #官方示例（可删除）
│  └─ markdown-examples.md # 官方示例（可删除）
├─ node_modules/  # vitepress依赖包（npm install生成，不需要上传）
├─ package-lock.json
└─ package.json
  1. 上传之后，你需要检查仓库Actions有没有正常运行

绿色勾是成功，黄色圆圈是等待中，红色叉叉是失败。这里因为文件deploy-pages.yml的代码不同需要自己调试，建议直接扔给AI

  1. 开启GitHub Pages服务：
Settings/Pages

Build and deployment下：

Source选择Deploy from a branch

Branch选择gh-pages，文件夹选/ (root)

等个一两分钟网址应该就出来了，记住不要选择Github Actions
